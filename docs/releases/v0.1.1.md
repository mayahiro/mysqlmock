# v0.1.1 Release Notes

`v0.1.1` is a bug fix release for schema dump compatibility.

## Bug Fixes

- Fixed schema loading when multiple tables use the same MySQL index name
- MySQL-visible index names remain table-scoped, while mysqlmock maps them to
  private SQLite index names internally to avoid SQLite's schema-wide index
  namespace
- `SHOW KEYS` and `information_schema.statistics` continue to expose the
  original MySQL index names
- `DROP INDEX`, `ALTER TABLE ... DROP INDEX`, and `ALTER TABLE ... RENAME INDEX`
  resolve MySQL index names to the internal SQLite index names

## Upgrade Notes

No config changes are required. Existing schema files that previously failed
because the same index name appeared on multiple tables should now load without
renaming those indexes.
