# v0.1.0 Release Notes

`v0.1.0` は mysqlmock の最初の tagged release です

mysqlmock は Go の Repository test 向けの軽量な MySQL protocol test server です
実際の MySQL client connection を受け付け、状態を持つ CRUD は SQLite backend で実行し、Repository test に必要な範囲の MySQL compatibility を提供します

## Highlights

- `github.com/mayahiro/mysqlmock/pkg/mysqlmock` 配下の public Go library API
- `serve`、`check`、`dump-unsupported-template`、`dump-config-schema` を持つ `mysqlmock` CLI
- `github.com/go-sql-driver/mysql` と `database/sql` の一般的な操作向け MySQL wire protocol support
- Repository test でよく使う parameter type の prepared statement 実行
- transaction、rollback、commit、savepoint
- generated JSON Schema output 付き YAML config file
- inline SQL または SQL dump file からの schema setup
- inline config、YAML、JSON、CSV file からの seed data
- fixed response、MySQL-like error、delay、disconnect のための SQL rules
- generated rule suggestion と安定した JSON snapshot を含む unsupported-query diagnostics
- Repository test schema 向けの MySQL/TiDB DDL compatibility
- `INSERT ... ON DUPLICATE KEY UPDATE`、`INSERT IGNORE`、`REPLACE INTO` compatibility
- `CONCAT`、`DATE_FORMAT`、`JSON_EXTRACT`、`JSON_UNQUOTE` などの common scalar function compatibility
- `SHOW FULL FIELDS`、`SHOW CREATE TABLE`、`SHOW KEYS`、一部の `information_schema` query 向け ActiveRecord-style schema introspection

## Compatibility Scope

| 領域 | 状態 |
| --- | --- |
| Go version | Go 1.25 以上 |
| Go SQL client | `database/sql` と `github.com/go-sql-driver/mysql` |
| Core operations | Ping、query、exec、prepared statement、transaction、savepoint |
| Schema setup | inline schema、SQL dump file、よく使う MySQL/TiDB DDL translation |
| Seed data | inline rows と YAML、JSON、CSV seed files |
| ORM behavior | よく使う GORM setup variables と ActiveRecord-style schema introspection |
| MySQL/TiDB compatibility | focused behavior comparison の対象。ただし full server replacement ではありません |
| 対象外 | 完全な MySQL SQL parser、optimizer behavior、本物の row lock、TLS、compression、`multiStatements=true`、`LOAD DATA LOCAL INFILE` |

## Known Limitations

- mysqlmock は production database ではなく、完全な MySQL compatibility を目標にしていません
- prepared statement support は、すべての MySQL binary protocol type を網羅していません
- `SET NAMES` は connection character set variable を記録しますが、query data や result data の transcoding は行いません
- TLS、compression、`multiStatements=true`、`LOAD DATA LOCAL INFILE` は未対応です
- `FOR UPDATE` locking clause は Repository test 向けに strip しますが、本物の row lock や lock conflict は再現しません
- MySQL-specific SQL compatibility は意図的に小さく保っており、実際の unsupported-query report から拡張する前提です

## Upgrade Notes

これは initial release のため、過去の tagged version からの migration step はありません
