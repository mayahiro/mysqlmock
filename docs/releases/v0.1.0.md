# v0.1.0 Release Notes

`v0.1.0` is the first tagged release of mysqlmock.

mysqlmock is a lightweight MySQL-protocol test server for Go tests that touch
database-backed code. It accepts real MySQL client connections and executes
stateful CRUD operations against a SQLite backend with focused MySQL
compatibility.
Use mysqlmock when you want faster database-backed tests than Dockerized MySQL,
but more realistic coverage than query expectation mocks.
It is useful not only for repository tests, but also for service-level tests
that should exercise real repository implementations without starting MySQL.

## Highlights

- Public Go library API under `github.com/mayahiro/mysqlmock/pkg/mysqlmock`
- `mysqlmock` CLI with `serve`, `check`, `dump-unsupported-template`, and
  `dump-config-schema`
- MySQL wire protocol support for common `database/sql` operations with
  `github.com/go-sql-driver/mysql`
- Prepared statement execution for common repository-test parameter types
- Transactions, rollback, commit, and savepoints
- YAML config files with generated JSON Schema output
- Schema setup from inline SQL or SQL dump files
- Seed data from inline config, YAML, JSON, or CSV files
- SQL rules for fixed responses, MySQL-like errors, delays, and disconnects
- Unsupported-query diagnostics with generated rule suggestions and stable JSON
  snapshots
- MySQL/TiDB DDL compatibility for common repository-test schemas
- `INSERT ... ON DUPLICATE KEY UPDATE`, `INSERT IGNORE`, and `REPLACE INTO`
  compatibility
- Common scalar function compatibility, including `CONCAT`, `DATE_FORMAT`,
  `JSON_EXTRACT`, and `JSON_UNQUOTE`
- ActiveRecord-style schema introspection for `SHOW FULL FIELDS`,
  `SHOW CREATE TABLE`, `SHOW KEYS`, and selected `information_schema` queries

## Compatibility Scope

| Area | Status |
| --- | --- |
| Go version | Go 1.25 or newer |
| Go SQL client | `database/sql` with `github.com/go-sql-driver/mysql` |
| Core operations | Ping, query, exec, prepared statements, transactions, savepoints |
| Schema setup | Inline schema, SQL dump files, common MySQL/TiDB DDL translation |
| Seed data | Inline rows and YAML, JSON, or CSV seed files |
| ORM behavior | Common GORM setup variables and ActiveRecord-style schema introspection |
| MySQL/TiDB compatibility | Focused behavior comparison target, not a full server replacement |
| Out of scope | Full MySQL SQL parsing, optimizer behavior, real row locks, TLS, compression, `multiStatements=true`, `LOAD DATA LOCAL INFILE` |

## Known Limitations

- mysqlmock is not a production database and does not aim for full MySQL
  compatibility
- Prepared statement support does not cover every MySQL binary protocol type
- `SET NAMES` records connection character set variables but does not transcode
  query or result data
- TLS, compression, `multiStatements=true`, and `LOAD DATA LOCAL INFILE` are
  not supported
- `FOR UPDATE` locking clauses are stripped for repository tests, but real row
  locks and lock conflicts are not emulated
- MySQL-specific SQL compatibility is intentionally small and should grow from
  real unsupported-query reports

## Upgrade Notes

This is the initial release, so there are no migration steps from a previous
tagged version.
